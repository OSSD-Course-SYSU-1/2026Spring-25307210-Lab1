# 2026Spring-25307210-Lab1
第一次作业：对本门课程的想法
对“开源软件开发技术”这门课程的期望，可以从以下两个核心维度来展开：
一、 夯实技术基础与工程实践能力
期望通过本课程系统掌握开源软件开发的完整工具链与核心技术。这不仅包括熟练使用 Git 进行版本控制、分支管理以及 Pull Request 等标准协作流程，更希望深入理解 CI/CD（持续集成/持续交付）、DevOps 自动化流程以及代码审查的最佳实践。期望能够打破传统封闭开发的思维局限，在真实的分布式团队协作环境中，学会如何编写规范、可维护且符合社区标准的代码，将理论知识转化为扎实的工程落地能力。
二、 深度融入开源文化与社区协作
除了硬核的技术技能，同样期望能深刻理解开源背后的哲学与协作文化。希望能学会如何在开放、透明的社区中高效沟通（如 Issue 讨论、异步交流），理解并遵守开源许可证协议与合规性要求。期望通过参与真实的开源项目（哪怕是修复文档或解决 Good First Issue），亲身体验从“使用者”到“贡献者”甚至“维护者”的角色转变，培养跨地域、跨文化的协同工作能力与开源责任感。
